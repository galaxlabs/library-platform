
### Step 2: Edit Environment (if needed)
```bash
nano backend/.env
# Update DB credentials if using different database
```

### Step 3: Setup Nginx (if not already done)
```bash
sudo cp /home/fg/library-platform/infra/nginx/library.conf \
    /etc/nginx/sites-available/library.conf
sudo ln -s /etc/nginx/sites-available/library.conf \
    /etc/nginx/sites-enabled/library.conf
sudo nginx -t
sudo systemctl reload nginx
```

### Step 4: Start Services
```bash
cd /home/fg/library-platform
bash start-live.sh
```

### Step 5: Verify Running
```bash
bash status-live.sh
```

Expected output:
```
✓ Backend (gunicorn) - Running on port 8001
✓ Frontend (Next.js) - Running on port 3000
✓ Nginx - Running (port 80/443)
```

## 🔧 How It Works

### Architecture
```
User Request (Port 80)
    ↓
Nginx Reverse Proxy
    ├─ / routes to Frontend (localhost:3000) ← Frontend First!
    ├─ /api/* routes to Backend (localhost:8001)
    └─ /admin/* routes to Admin (localhost:8001)
```

### Service Architecture
```
Frontend                Backend               Database
(Next.js)              (Django)              (PostgreSQL)
  ↓ API calls ↓          ↓ queries ↓
Port 3000          Port 8001            Local/Network
```

### Services Won't Start?

### Nginx Not Routing?
